# 改造路线图（初版）

## 阶段1：文档与审计
- 完成架构/机制文档与债务清单
- 插入非侵入注释（物理/AI/玩家主循环关键段）
- 引入基础性能采样与日志格式

## 阶段2：平台与渲染现代化
- 切换 URP；统一光照/材质；清理旧后处理
- Addressables 或 YooAssets 统一资源加载
- 输入系统评估：保留 InControl 或迁移 Unity 新输入

## 阶段3：物理与并发升级
- 将 `SimulationThread` 的关键计算迁至 Job + Burst（先从样条/力积分热点开始）
- 用 `Unity.Mathematics` 替换 `Mono.Simd`；封装线程安全动作队列
- 建立仿真基准测试与回归

## 阶段4：玩家控制器解耦
- 拆分 `PlayerController` 为：`PlayerInput`/`PlayerFSM`/`PlayerAnim`/`PlayerPhysicsBridge`/`PlayerUI`
- 引入 ScriptableObject 配置（阈值/速度/概率等）

## 阶段5：AI系统进化
- 行为树/配置驱动；生成咬钩模板数据资产
- 增强避障与环境响应（流速/波浪/时间/天气）

## 阶段6：玩法与经济
- 任务/活动/排名；经济参数与防刷策略；数值回归测试

## 阶段7：网络/社交（可选）
- 方案A：实时房间（Photon Fusion）用于钓赛
- 方案B：异步社交（服务器记分与分享）

## 阶段8：QA与发布
- 自动化测试覆盖、设备分档与帧率策略
